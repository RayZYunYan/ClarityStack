# ClarityStack Style Guide

## Voice

- Clear, informed, and practical.
- Helpful to builders, engineers, and technically curious readers.
- Calm confidence over hype.

## Editorial principles

- Start with the most important idea or implication.
- Preserve technical accuracy and cite original sources.
- Prefer concrete examples over vague trend language.
- Explain why the item matters for practitioners.

## Platform notes

### LinkedIn

- Professional and readable.
- Keep the post compact and conversational.
- Use 2-3 emoji at most when they improve clarity.

### Blog

- Use descriptive section headers.
- Add short code snippets only when they clarify a technical point.
- End with a references section listing original URLs.

### X

- Lead with a hook.
- Keep each post self-contained and sharp.
- Avoid filler so every character earns its place.
